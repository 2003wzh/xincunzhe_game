# Codex 工作规则

本项目由个人开发者和 Codex 协作完成。Codex 在本仓库工作时必须遵守以下规则。

## 项目定位

- 项目名：XianxiaSurvivor
- 类型：Unity 2D 单机修仙幸存者
- 目标平台：PC
- 第一阶段目标：15 分钟可玩的 MVP Demo

## 当前阶段范围

MVP 只做：

1. 玩家移动
2. 怪物刷出并追踪玩家
3. 玩家自动释放功法攻击
4. 怪物死亡掉落灵气
5. 玩家吸收灵气升级
6. 升级时弹出三选一功法
7. 15 分钟出现 Boss
8. 玩家死亡失败，击败 Boss 通关
9. 局外简单境界提升

不要主动扩展到联机、开放世界、背包、复杂剧情、炼丹、炼器或宗门经营。

## 修改前规则

在修改代码前，Codex 必须先说明：

1. 实现计划
2. 准备新增的文件
3. 准备修改的文件

得到确认后再实际修改。

## 修改代码规则

- 每次只完成用户指定的模块。
- 不要一次性重写整个项目。
- C# 代码保持简单清楚，适合新手维护。
- 每个类或接口前写清楚用途。
- 不要把所有逻辑写进一个大脚本。
- UI 和玩法逻辑尽量通过事件解耦。
- 怪物、子弹、掉落物等频繁生成对象必须考虑对象池。
- 避免在 `Update` 中频繁调用 `FindObjectOfType` 或类似查找。
- 避免运行时频繁 `Instantiate` 和 `Destroy`。
- 不引入第三方插件，除非用户明确确认。

## 目录规则

- `Assets/Scripts/Core`：游戏状态、计时、事件
- `Assets/Scripts/Player`：玩家
- `Assets/Scripts/Combat`：战斗通用接口和数据
- `Assets/Scripts/Enemies`：怪物和 Boss
- `Assets/Scripts/Skills`：功法和升级选择
- `Assets/Scripts/Data`：配置数据
- `Assets/Scripts/UI`：界面
- `Assets/Scripts/Utils`：通用工具

## 修改后规则

每次修改完成后，Codex 必须说明：

1. 实际新增了哪些目录
2. 实际新增了哪些文件
3. 每个文件的作用
4. 如何在 Unity 中测试
5. 是否需要用户手动操作
